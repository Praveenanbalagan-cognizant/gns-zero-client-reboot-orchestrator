# Submission Checklist

## Must-Have Files

- [ ] `README.md` with setup instructions.
- [ ] `architecture.md` with agent architecture.
- [ ] `summary.md` with 1 to 2 page project summary.
- [ ] `requirements.txt` with required packages.
- [ ] Source code for AP reboot logic.
- [ ] Neuro SAN registry or agent configuration.
- [ ] Synthetic sample data only.
- [ ] Short demo video link, maximum 5 minutes.

## Clean Repo Rules

- [ ] No `.venv` or `venv` folder.
- [ ] No `.env` file.
- [ ] No API keys.
- [ ] No client data.
- [ ] No production AP inventory.
- [ ] No real controller screenshots.
- [ ] No internal logs.
- [ ] No unnecessary copied Neuro SAN Studio files.
- [ ] No large generated output unless required for demo.

## Scoring Strategy

| Scoring Area | Weight | What To Show |
|---|---:|---|
| Novelty of concept | 35% | Real wireless operations use case, safe AP reboot decisioning, closed-loop validation, audit-ready report. |
| Neuro SAN features | 35% | Multiple specialist agents, front-agent orchestration, coded tool integration, delegation, final synthesis. |
| Code quality | 30% | Clean folder structure, synthetic data, dry-run default, clear guardrails, small focused functions, readable docs. |
| Video | 20 marks | Clear problem, architecture, live demo, safety, and why Neuro SAN is needed. |

## Winning Demo Flow

1. State the real operational problem.
2. Show the agent architecture.
3. Run a dry-run AP reboot plan.
4. Highlight blocked APs and explain why they were blocked.
5. Show validation and final report.
6. Explain how production integration would be safely added.

## Final Submission Message

Use this short text in the submission form:

`GNS Zero-Client Reboot Orchestrator is a Neuro SAN based wireless operations assistant that safely identifies APs with high uptime and zero clients, applies risk guardrails, performs dry-run reboot planning, validates recovery, and generates an audit-ready report using only synthetic demo data.`
